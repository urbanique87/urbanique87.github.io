## Next.js는 무엇인가요?

Next.js 공식 문서에 작성된 <a href="https://nextjs.org/docs#what-is-nextjs" target="_blank" rel="noopener noreferrer">What is Next.js?</a>를 살펴보겠습니다.

> Next.js is a React framework for building full-stack web applications.
> You use React Components to build user interfaces, and Next.js for additional features and optimizations.
> 
> Under the hood, Next.js also abstracts and automatically configures tooling needed for React, like bundling, compiling, and more.
> This allows you to focus on building your application instead of spending time with configuration.
> 
> Whether you're an individual developer or part of a larger team, Next.js can help you build interactive, dynamic, and fast React applications.
<br />
Next.js는 풀스택 웹 애플리케이션을 구축하기 위한 React 프레임워크입니다.
React 컴포넌트를 사용하여 사용자 인터페이스를 구축하고, 추가 기능과 최적화를 위해 Next.js를 사용합니다.

내부적으로, Next.js는 React에 필요한 번들링, 컴파일링 등의 도구를 추상화하고 자동으로 구성합니다.
이를 통해 설정에 시간을 들이지 않고, 애플리케이션을 구축하는데 집중할 수 있습니다.

개인 개발자 또는 대규모 팀의 일원이든, Next.js는 인터랙티브하고, 동적이며, 빠른 React 애플리케이션을 구축하는 데 도움이 될 수 있습니다.
<br />
## Next.js의 인터랙티브하고 동적인 특징이란?
Next.js 공식 문서에서는 "인터랙티브하고 동적이며 빠른 애플리케이션"을 구축할 수 있다고 소개합니다. 이러한 특징은 Next.js만의 특징이라기보다는 많은 현대 웹 프레임워크들이 제공하는 기능입니다. 

그러나 Next.js는 특히 그 구현과 최적화 측면에서 차별화된 점이 있습니다. 결국, 이는 단순한 마케팅 문구가 아닌, Next.js가 제공하는 핵심적인 기술적 특징들을 함축적으로 표현한 것입니다.

<br />
## Next.js에서 제공하는 강력한 기능
Next.js에서 특별하게 제공되는 강력한 기능은 다음과 같습니다.

<br />
### 1. 서버 사이드 렌더링(SSR)과 동적 콘텐츠
SSR은 단순히 서버에서 HTML을 생성하는 것을 넘어, 다음과 같은 동적 기능을 제공합니다.

**하이드레이션(Hydration)과 동적 렌더링**

- 서버에서 생성된 정적 HTML에 React 이벤트 핸들러와 상태 관리 기능이 클라이언트 측에서 자동으로 연결되어 상호작용이 가능한 애플리케이션으로 전환됩니다.
- SEO 최적화와 초기 로딩 성능이 향상됩니다.

**실시간 데이터 처리**

- 서버 컴포넌트와 API 라우트를 활용하여 실시간 데이터 페칭이 가능하며, SWR이나 React Query와 같은 데이터 관리 도구와 통합하여 효율적인 상태 동기화가 가능합니다.
- 사용자 상호작용(폼 제출, 클릭 등)에 따른 동적 데이터 처리가 가능합니다.
- 서버 컴포넌트를 통한 효율적인 데이터 처리가 가능합니다.

<br />
### 2. 정적 사이트 생성(SSG)과 동적 업데이트

Next.js는 SSG를 통해 초기 페이지 로딩 성능을 높이면서도 동적인 콘텐츠를 처리할 수 있습니다.

**하이브리드 렌더링 (Hybrid Rendering)**

- 빌드 시점에 페이지를 미리 생성하여 최고의 성능을 제공합니다.
- 정적 페이지와 동적 기능을 효과적으로 조합할 수 있습니다.
- 예: 게시글 목록은 정적으로 생성하고, 추가 게시글은 동적으로 불러 올 수 있습니다.

**ISR(Incremental Static Regeneration)**

- 설정된 재생성 주기나 on-demand 요청에 따라 백그라운드에서 정적 페이지를 자동으로 재생성하여, 항상 최신 콘텐츠를 제공하면서도 정적 페이지의 성능상 이점을 유지합니다.
- 콘텐츠 업데이트가 필요한 경우에만 페이지를 생성합니다.
- 캐싱과 동적 업데이트의 장점을 모두 활용할 수 있습니다.

<br />
### 3. React와의 결합: 향상된 언터랙티비티

Next.js는 React의 강력한 기능을 확장하여 더욱 풍부한 인터랙션을 제공합니다.

**상태 관리와 컴포넌트 기반 설계**

- React의 상태 관리와 컴포넌트 기반 설계를 그대로 활용할 수 있습니다.
- 사용자 입력에 따른 실시간 UI 업데이트를 구현할 수 있습니다.

**서버 컴포넌트 통합**

- 서버에서 실행되는 컴포넌트로 초기 로딩 성능을 향상시킵니다.
- 클라이언트 컴포넌트와 자연스럽게 조화를 이룹니다.
- 데이터 페칭과 상태 관리를 효율적으로 처리할 수 있습니다.
- 서버 컴포넌트는 클라이언트 번들 크기를 줄이고, 데이터베이스나 파일 시스템에 직접 접근할 수 있어 더 효율적인 데이터 처리가 가능합니다. 또한 민감한 정보를 클라이언트에 노출하지 않고도 안전하게 처리할 수 있습니다.

<br />
### 4. 빠른 반응성과 자동 최적화

사용자 경험(UX)를 향상시키는 자동 최적화 기능을 제공합니다.

**코드 분할**

- 페이지별로 필요한 최소한의 코드만 로드합니다.
- 페이지 성능을 최적화하여 불필요한 코드 로딩을 방지합니다.

**효율적인 자원 관리**

- Next.js의 Image 컴포넌트를 통한 자동 이미지 최적화, 폰트 최적화, 그리고 스크립트 최적화 기능은 Core Web Vitals 지표를 개선하는 데 크게 기여합니다.
- 필요한 시점에 필요한 리소스만 로드하여 성능을 개선합니다.
- 개발자가 별도의 설정 없이도 최적화된 결과를 얻을 수 있습니다.

<br />
### 5. 동적 라우팅과 네비게이션

**파일 시스템 기반 라우팅**(File System Based Routing)

- 직관적인 페이지 구조를 제공합니다.
- 동적 라우트 설정과 페이지 전환을 지원합니다.
- 페이지 접근 시 필요한 데이터를 동적으로 불러와 렌더링합니다.

**클라이언트 사이드 네비게이션**

- 페이지 전환 시 부드러운 사용자 경험을 제공합니다.
- 자동 prefetching으로 빠른 페이지 로딩을 지원합니다.
- 동적 라우트 파라미터를 활용한 유연한 페이지 구성이 가능합니다.

<br />
## 결론

Next.js가 **인터랙티브하고 동적** 이라고 강조하는 이유는 SSR(서버 사이드 렌더링)과 SSG(정적 사이트 생성), ISR(증분 정적 재생성) 등 다양한 렌더링 전략을 통해 사용자와의 상호작용에 따라 실시간으로 콘텐츠를 처리할 수 있기 때문입니다.

이는 웹 애플리케이션의 성능과 사용자 경험(UX)을 최적화하는 데 중요한 역할을 합니다.


<br />
## 용어 설명


- **SSR (Server-Side Rendering)**: 서버에서 페이지를 렌더링하여 클라이언트로 전송하는 방식입니다.
- **SSG (Static Site Generation)**: 빌드 시점에 페이지를 미리 생성하는 방식입니다.
- **ISR (Incremental Static Regenration)**: 정적 페이지를 주기적으로 재생성하는 기능입니다.
- **하이드레이션 (Hydration)**: 서버에서 생성된 HTML이 클라이언트에서 완전한 React 애플리케이션으로 활성화되는 과정입니다.
- **서버 컴포넌트 (Server Components)**: 서버에서 실행되어 클라이언트로 전달되는 React 컴포넌트입니다.
